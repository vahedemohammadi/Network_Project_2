# Phase 2 Report

## Network Project 2

### HTTP Header and TCP/IP Protocol Stack Analysis

---

## Objective

The objective of Phase 2 is to analyze the protocol headers of a captured HTTP packet using Wireshark. The packet is examined at the Application, Transport, and Network layers of the TCP/IP protocol stack.

---

## Display Filter

The following Wireshark display filter was used to locate the HTTP packets:

```text
http
```

Alternatively, the following filter can also be used:

```text
tcp.port == 80
```

---

## Selected Packet

The first HTTP **GET** request generated by the **cURL** command was selected for analysis.

---

# Application Layer (HTTP)

### Extracted Information

| Field | Value |
|------|------|
| HTTP Method | GET |
| HTTP Version | HTTP/1.1 |
| Host | neverssl.com |
| User-Agent | curl/8.13.0 |

### Analysis

The packet contains an HTTP GET request sent by the cURL application. The request uses HTTP/1.1 and specifies **neverssl.com** in the Host header. The User-Agent header identifies the client software as **curl version 8.13.0**.

---

# Transport Layer (TCP)

### Extracted Information

| Field | Value |
|------|------|
| Protocol | TCP |
| Source Port | 10840 |
| Destination Port | 80 |

### Analysis

The communication uses the Transmission Control Protocol (TCP). The source port (**10840**) is an ephemeral port automatically assigned by the operating system for this client connection. The destination port (**80**) is the default port for HTTP services.

TCP provides reliable communication by ensuring packet delivery, ordering, and error checking between the client and the web server.

---

# Network Layer (IPv4)

### Extracted Information

| Field | Value |
|------|------|
| Source Address | 192.168.1.12 |
| Destination Address | 34.223.124.45 |

### Analysis

The source IP address belongs to the local client machine, while the destination IP address belongs to the remote web server hosting **neverssl.com**. The IPv4 protocol is responsible for routing packets between the sender and the destination across the network.

---

# Packet Encapsulation

The captured packet demonstrates the encapsulation process in the TCP/IP protocol stack:

1. The HTTP request is created at the Application layer.
2. The request is encapsulated inside a TCP segment.
3. The TCP segment is encapsulated inside an IPv4 packet.
4. The packet is transmitted over the network to the destination web server.

In this phase, the HTTP request packet was successfully analyzed using Wireshark. Important header fields from the Application, Transport, and Network layers were identified and examined.
