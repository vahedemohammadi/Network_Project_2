### HTTP Header and TCP/IP Protocol Stack Analysis

# Objective

The objective of Phase 2 is to analyze the protocol headers of a captured HTTP packet using **Wireshark**. In this phase, the selected packet is examined across the three main layers of the **TCP/IP protocol stack**: the **Application Layer**, **Transport Layer**, and **Network Layer**.

---

# Display Filters

The following Wireshark display filter was used to locate HTTP packets:

```text
http
```

Alternatively, the following filter can also be used to display HTTP-related traffic:

```text
tcp.port == 80
```

---

# Selected Packet

For this analysis, the first **HTTP GET** request generated by **cURL** was selected. This packet initiates the communication between the client and the web server and contains important protocol header information for analysis.

---

# Application Layer Analysis (HTTP)

## Extracted Information

| Field | Value |
|-------|-------|
| HTTP Method | GET |
| HTTP Version | HTTP/1.1 |
| Host | neverssl.com |
| User-Agent | curl/8.13.0 |

## Analysis

The selected packet contains an **HTTP GET** request generated by **cURL**.

The request uses **HTTP/1.1**, specifies the destination domain through the **Host** header (**neverssl.com**), and identifies the client software using the **User-Agent** header (**cURL 8.13.0**).

The **GET** method is used to retrieve resources from a web server. Unlike methods such as **POST**, it does not modify server-side resources. In this request, the client simply asks the server to return the requested web page.

---

# Transport Layer Analysis (TCP)

## Extracted Information

| Field | Value |
|-------|-------|
| Protocol | TCP |
| Source Port | 10840 |
| Destination Port | 80 |

## Analysis

The communication uses the **Transmission Control Protocol (TCP)**.

The source port (**10840**) is an **ephemeral port** automatically assigned by the operating system for this connection. The destination port (**80**) is the standard port used for **HTTP** communication.

TCP provides reliable communication between the client and the server. It ensures that data is delivered accurately, completely, and in the correct order by using sequence numbers, acknowledgments, error detection, flow control, and retransmission of lost packets.

---

# Network Layer Analysis (IPv4)

## Extracted Information

| Field | Value |
|-------|-------|
| Source IP Address | 192.168.1.12 |
| Destination IP Address | 34.223.124.45 |

## Analysis

The source IP address (**192.168.1.12**) belongs to the client computer within the local network.

The destination IP address (**34.223.124.45**) belongs to the public web server hosting **neverssl.com**.

At this layer, the **IPv4** protocol is responsible for logical addressing and packet routing across the network. Based on these IP addresses, routers forward the packet through intermediate networks until it reaches the destination server.

---

# Packet Encapsulation

The captured packet clearly demonstrates the encapsulation process within the **TCP/IP protocol stack**.

The process occurs as follows:

1. An **HTTP** request is created at the **Application Layer**.
2. The HTTP message is encapsulated inside a **TCP segment**.
3. The TCP segment is encapsulated inside an **IPv4 packet**.
4. The IP packet is transmitted across the network toward the destination server.

This layered architecture illustrates how each protocol layer treats the data from the upper layer as its payload while adding its own protocol header before forwarding the packet to the next lower layer.
